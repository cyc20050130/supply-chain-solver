# 供应链表格直接出结果规则

## 1. 目标

本规则用于处理 `C:\Users\cyc20\Desktop\供应链管理` 目录下及后续新增的供应链 `.xls` 表格。

核心目标不是解释过程，而是实现以下工作方式：

> 给我一个 `.xls` 表格，我直接输出结果。

输出结果只保留你关心的内容：

- 销售量
- 生产量
- 采购量
- 运输计划
- 总分

并且：

- 固定按 **30 天周期** 求解
- 销量预测采用 **移动加权平均法**
- 采用 **启发式算法 / 规则工作流**
- **不使用 MLP**
- 对单一模块题，**只输出能够可靠推导的部分**，不强行补全缺失链条
- 最终结果要方便查阅与归档

---

## 2. 文件输出约定

### 2.1 每个 xls 对应一个结果文件夹

当输入一个 `xxx.xls` 时，生成一个与其**同名（去掉扩展名）**的结果文件夹：

```text
原始文件：xxx.xls
结果文件夹：xxx\
```

这样可以避免与原始 `.xls` 文件重名冲突。

### 2.2 结果文件夹内容

结果文件夹内至少放：

```text
xxx\
├─ xxx.xls
└─ 方案.md
```

说明：

- `xxx.xls`：将原始表格复制到结果文件夹中，方便对照
- `方案.md`：写入最终结果表与总分

如后续需要，也可以增补：

- `结果表.csv`
- `提取字段.json`
- `校验结果.md`

但对外默认只看：

- 原始 `xls`
- `方案.md`

### 2.3 方案.md 输出风格

`方案.md` 默认采用**极简结果表**，不写长篇推导，不展开模型细节。

只保留：

- 销售量表
- 生产量表
- 采购量表
- 运输计划表
- 总分

## 3. 题型识别规则

先识别表格属于以下哪一类：

1. **采购题**
2. **生产题**
3. **销售题**
4. **综合题**

识别后按对应工作流处理。

### 3.1 采购题
通常重点字段包括：

- 供应商
- 原料单价
- 供应商可供货量
- 原料运输路线
- 提前期
- 起运量 / 起运费
- 工厂原料库存
- 生产消耗或生产计划

输出重点：

- 采购量
- 原料运输计划
- 总分

如题内已给生产消耗或可推出生产量，也可附带生产量。

### 3.2 生产题
通常重点字段包括：

- 工厂产能
- 成品库存
- 销售网点需求或历史销量
- BOM
- 原料库存
- 成品运输路线

输出重点：

- 销售量
- 生产量
- 成品运输计划
- 若 BOM 和原料库存清楚，则补充采购量
- 总分

### 3.3 销售题
通常重点字段包括：

- 销售网点历史销量
- 库存上限
- 期初库存
- 补货路线
- 提前期
- 承运商费率

输出重点：

- 销售量
- 发货 / 补货量
- 运输计划
- 总分

### 3.4 综合题
通常同时包含：

- 销售预测
- 生产计划
- 原料采购
- 原料运输
- 成品运输
- 多项评分指标

输出重点：

- 销售量
- 生产量
- 采购量
- 原料运输计划
- 成品运输计划
- 总分

## 4. 通用启发式求解总纲

## 4.1 固定周期

统一以 **30 天** 为计划期。

所有销量、生产量、采购量、运输量，最终都折算到 30 天口径。

## 4.2 销量预测规则

默认采用 **移动加权平均法**。

### 标准口径
若表格提供最近 6 期历史销量，则采用：

```text
权重 = 1:2:3:4:5:6
```

越近的历史期权重越高。

### 30 天销量预测公式

若最近 6 期销量分别为：

```text
S1, S2, S3, S4, S5, S6
```

则加权平均销量为：

```text
(1*S1 + 2*S2 + 3*S3 + 4*S4 + 5*S5 + 6*S6) / 21
```

再根据表格时间口径折算为 30 天需求。

### 补充规则

- 若表格直接给出 30 天逐日销量数据，则优先汇总 / 平滑后转为 30 天需求
- 若题目已有明确需求量，则不重复预测，直接采用题目需求
- 若综合题存在“预测偏差率”考核，则必须保留预测值与实际值的对照字段

## 4.3 多工厂分配规则

如果有多家工厂，先按以下顺序分配每个代理 / 门店的供货工厂：

1. **地理位置最近**
2. **可达路线更短、提前期更小**
3. **单位物流成本更低**
4. **工厂产能更充足**
5. **工厂库存更能直接支撑**

先完成“销售网点 -> 工厂”的归属，再汇总每个工厂的 30 天总需求。

## 4.4 安全库存规则

安全库存用于防止断货。

默认依据销售波动确定，而不是固定拍值。

### 推荐启发式

先计算：

- 日均销量
- 日均销量波动（方差或标准差）

再按波动大小给出安全库存天数：

- 波动低：1 天安全库存
- 波动中：2 天安全库存
- 波动高：3 天安全库存
- 若题目提前期长、运输不稳定，可上调到 4~5 天

### 默认安全库存公式

```text
建议安全库存 = 日均销量 × 安全库存天数
```

如题目已有明确库存上限，则安全库存不能把期末库存推到上限以上。

## 4.5 生产量计算规则

对每个工厂：

```text
建议生产量 = 分配到该工厂的30天需求 + 目标安全库存 - 成品期初库存 - 已知在途成品
```

并同时满足：

- 不超过 30 天总产能
- 尽量避免期末库存过高
- 优先保证满足率，再压低成本

如果成品库存已经足够覆盖 30 天需求，则生产量可以为 0 或很低。

## 4.6 原料需求与采购量规则

若题目提供 BOM，则对每种原料：

```text
30天原料消耗 = 生产量 × BOM 用量
```

```text
建议采购量 = 30天原料消耗 + 原料安全库存 - 原料期初库存 - 已知在途原料
```

若期初原料库存已足够支撑生产，则采购量可为 0。

## 4.7 供应商选择规则

采购不能只看单价，必须看**到厂总成本**与**供货可行性**。

### 供应商排序指标

按以下顺序综合判断：

1. 原料单价
2. 原料运输成本
3. 起运量 / 起运费要求
4. 提前期是否能赶上生产
5. 供应商可供货量是否足够
6. 到货后是否会顶爆工厂库存上限

### 默认分配逻辑

- 先用**落地总成本最低**且**满足提前期**的供应商
- 当其可供货量不足时，再由次优供应商补足
- 国际采购若虽然便宜，但提前期过长、港口堆存风险高，则优先级下调

## 4.8 运输方式选择规则

运输目标：

> 在满足生产消耗或销售需求的前提下，尽可能降低物流成本。

### 运输方式启发式

#### 优先铁路 / 海运联运的情形

- 货量大
- 达到起运量门槛
- 干线距离长
- 单位综合运价明显低于汽运

#### 优先直达汽运的情形

- 货量小
- 未达到铁路 / 海运最低起运要求
- 无合适中转线路
- 时效要求高
- 中转会引入堆存费 / 超限费

## 4.9 批次拆分规则

不能只给总运输量，必须考虑：

- 提前期
- 工厂 / 门店库存上限
- 节点堆存费
- 超限堆存费

因此默认规则是：

- **大批量但库存有限**：拆成多趟发运
- **到货过早会挤压库容**：推迟首发时间或拉大发运间隔
- **同一路线、同一承运商、同一单趟量、固定间隔发运**：可以合并成一条录入计划

运输计划默认要给出：

- 起点
- 终点
- 路线
- 承运商
- 运输方式
- 单趟运量
- 起运日期
- 承运趟数
- 发运间隔
- 提前期

## 4.10 必做校验

在输出最终结果前，至少校验：

1. 供应商供货量不为负且不超供货上限
2. 工厂原料库存不为负
3. 工厂产量不超产能
4. 工厂成品库存不为负
5. 销售网点库存不为负
6. 市场 / 生产满足率达标情况明确
7. 运输路线真实存在
8. 起运量规则未被违反
9. 批次到货时间不晚于需求时点
10. 采购量、生产量、运输量前后配平
11. 若结果涉及离散单位，则做整数化

## 6. 评分规则

## 6.1 采购题评分规则（按评分图）

总分：

```text
分数 = 单位采购成本分数 + 生产满足率分数
```

### 1）单位采购成本分数

```text
单位采购成本分数 =
(1 - (我的单位采购成本 - 单位采购成本指标) / (单位采购成本指标 × 20%)) × 60
```

规则：

- 最高 60 分
- 最低 0 分
- 若低于或优于指标，则按上限封顶，不超过 60 分
- 若明显劣于指标，则按下限封底，不低于 0 分

### 2）生产满足率分数

```text
生产满足率分数 = (1 - (生产满足率指标 - 我的生产满足率)) × 40
```

规则：

- 达标得 40 分
- 未达标时按差额线性扣减
- 最低记 0 分

### 采购题求解目标

采购题优先级：

1. 先保证生产满足率
2. 再压低单位采购成本
3. 同时避免因为过量采购导致库存和运费恶化

## 6.2 综合题评分规则（按评分图）

总分：

```text
分数 = 预测偏差率分数 + 单位物流成本分数 + 单位采购成本分数 + 生产满足率分数 + 市场满足率分数
```

### 1）预测偏差率分数

```text
预测偏差率分数 = (1 - (实际预测偏差 / 预测偏差指标)) × 10
```

规则：

- 最高 10 分
- 最低 0 分
- 当偏差大于或等于指标值时，记 0 分

### 2）单位物流成本分数

```text
单位物流成本分数 =
(1 - (我的单位物流成本 - 单位物流成本指标) / (单位物流成本指标 × 20%)) × 20
```

规则：

- 最高 20 分
- 最低 0 分

### 3）单位采购成本分数

```text
单位采购成本分数 =
(1 - (我的单位采购成本 - 单位采购成本指标) / (单位采购成本指标 × 20%)) × 20
```

规则：

- 最高 20 分
- 最低 0 分

### 4）生产满足率分数

```text
生产满足率分数 = (1 - (生产满足率指标 - 我的生产满足率)) × 15
```

规则：

- 达标得 15 分
- 未达标线性扣减
- 最低 0 分

### 5）市场满足率分数

```text
市场满足率分数 = (1 - (市场满足率指标 - 我的市场满足率)) × 35
```

规则：

- 达标得 35 分
- 未达标线性扣减
- 最低 0 分

### 综合题求解目标

综合题优先级：

1. 先把预测偏差率压到可接受范围
2. 先保市场满足率与生产满足率
3. 再优化单位物流成本
4. 再优化单位采购成本
5. 最后收紧库存与批次安排，减少堆存和超限费

## 6.3 生产题临时评分规则（在正式评分图给出前使用）

生产题先采用**内部启发式总分**，只用于结果排序和给出一个总分，不代表平台最终分数。

```text
总分 = 需求满足得分 + 产能利用得分 + 成品库存合理性得分 + 运输可执行性得分
```

建议满分结构：

- 需求满足得分：50
- 产能利用得分：15
- 成品库存合理性得分：15
- 运输可执行性得分：20

### 临时打分原则

- 需求全部满足：接近满分
- 产量明显超产能：严重扣分
- 成品期末库存过高或过低：扣分
- 运输方案不可执行、提前期不够、起运量违规：扣分
- 路线过贵但有更优可行替代：扣分

## 6.4 销售题临时评分规则（在正式评分图给出前使用）

销售题先采用**内部启发式总分**，只用于结果排序和给出一个总分，不代表平台最终分数。

```text
总分 = 市场满足得分 + 预测合理性得分 + 库存合理性得分 + 物流可执行性得分
```

建议满分结构：

- 市场满足得分：50
- 预测合理性得分：20
- 库存合理性得分：15
- 物流可执行性得分：15

### 临时打分原则

- 市场满足率越高越好
- 预测偏差越小越好
- 期末库存不爆仓、不缺货越好
- 运输计划满足时效且成本更低越好

## 9. 执行原则总结

以后处理新表格时，遵循以下顺序：

1. 识别题型
2. 提取关键字段
3. 固定转成 30 天口径
4. 用移动加权平均法做销量预测
5. 依据地理位置与成本做工厂归属
6. 计算生产量
7. 计算采购量
8. 设计运输计划
9. 按对应规则计算总分
10. 生成同名结果文件夹，放入 `xls` 与 `方案.md`

本规则的核心是：

> **给表格，直接出结果。**

不是做 MLP，不是做花哨建模，而是用稳定、可解释、可复现的启发式工作流，快速得到可填、可看、可对比的答案。
